Optional errmsg

errmsg?: string
  • Defined in node_modules/@types/mongodb/index.d.ts:409

While not documented, the errmsg prop is AFAIK the only way to find out which unique index caused a duplicate key error. When you have multiple unique indexes on a collection, knowing which index caused a duplicate key error enables you to send better (more precise) error messages to the client/user (eg. "Email address must be unique" instead of "Both email address and username must be unique") â€“ which caters for a better (app) user experience.

Details: How to get index name on duplicate document 11000 error? (issue for mongoose, but the same applies for the native mongodb driver).

Note that in mongoose (the link above) the prop in question is called 'message' while in mongodb it is called 'errmsg'. This can be seen in multiple places in the source code, for example here.
